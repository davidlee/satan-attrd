# DR-001 – Lint cleanup: make tree GREEN under POL-001 strict clippy

## 1. Executive Summary

- **Delta**: [DE-001](./DE-001.md)
- **Status**: approved
- **Synopsis**: No architectural changes. Pure mechanical lint-compliance pass against POL-001's strict clippy config. All fixes are local: add doc sections, `zip` over parallel arrays, add reasoned `#[expect]` annotations, add `type` aliases for complex query tuples.

## 2. Problem & Constraints

- **Current Behaviour**: ~40 clippy violations fire under POL-001's strict config. Build is RED.
- **Drivers / Inputs**: POL-001 (`deny`-level lints across safety, correctness, style). `LINT-CLEANUP-HANDOFF.md` enumerates all violations.
- **Constraints**: No config changes. No business logic changes. `#[expect]` not `#[allow]`.
- **Out of Scope**: Test execution, migration code, lint-level changes.

## 3. Architecture Intent

- **Target Outcomes**: `just lint` silent, `cargo fmt --all --check` passes.
- **Guiding Principles**: Fix in spirit first; suppress only when lint is wrong here. Narrowest scope. Every suppression carries a `reason`.
- **State Transitions / Lifecycle Impact**: None — no runtime changes.

## 6. Design Decisions & Trade-offs

- **No architectural decisions.** Every fix is a local lint-compliance change. The design of the code does not change.
- **`#[expect]` over `#[allow]`**: Policy-mandated (POL-001 §4). Self-cleaning — stale suppressions error.
- **Zip over enumerate+index**: Cleaner, safer, makes the parallel-array invariant explicit in the type system.
