## 1. Summary & Context

- **Policy**: POL-001 – Rust lint policy for agentic development (strict clippy, zero warnings)
- **Implementation Plan**: [IP-001](./IP-001.md)
- **Change Drivers**: POL-001 installed the strict lint config. The build is intentionally RED — this delta makes it GREEN.

## 2. Motivation

POL-001 (`Cargo.toml [lints.rust]` / `[lints.clippy]` + `clippy.toml`, derived from forgettable's set) was installed. The tree currently has ~40 clippy violations across src/ and tests/. This delta resolves every one in the spirit of the lint, with narrow `#[expect]` suppressions only where the lint is genuinely wrong for that site.

## 3. Scope & Objectives

- **Primary Outcomes**:
  - `just lint` passes with zero warnings (gate: `cargo clippy --all-targets --all-features -- -D warnings`)
  - `cargo fmt --all --check` passes
  - Every suppression is narrow, reasoned, and uses `#[expect]` per POL-001 §4–6
- **Operational Constraints**: No config changes to `Cargo.toml [lints.*]` or `clippy.toml`. No business logic changes.
- **Dependencies**: None.

## 4. Out of Scope

- `just check` / tests (require Postgres on `$DATABASE_URL` — not available)
- Editing lint config levels
- Business logic or test-behaviour changes
- Migration code

## 5. Approach Overview

- **System Touchpoints**: All `src/*.rs` files, `tests/*.rs` integration tests, `Cargo.toml` [package] metadata
- **Key Changes**:
  1. `Cargo.toml` [package] — add `repository`, `keywords`, `categories` (fixes cargo-group lints)
  2. Test modules — convert `#[allow]` → `#[expect]` with reasoned lints
  3. doc comments — backtick bare identifiers (`doc_markdown`), add `# Errors` / `# Panics` sections
  4. `indexing_slicing` in real source — `zip` over parallel arrays, `.as_object_mut().insert()`
  5. `pub_use` — reasoned `#[expect]` on crate-facade re-exports
  6. `module_name_repetitions` — reasoned `#[expect]` per idiomatic item
  7. `type_complexity` — `type` aliases for `query_as` tuple types
  8. Various one-offs (`clone_on_copy`, `or_default`, `map_err_ignore`, etc.)
- **Second wave**: tests/*.rs lints surface only after src/ is clean

## 6. Verification Strategy

- **Gate**: `just lint` silent (zero warnings) + `cargo fmt --all --check` passes
- **Final-summary disclosure**: List every `#[expect]` added per POL-001 §Verification
- **No config/lint-level changes**

## 7. Follow-ups & Tracking

- **Future**: None anticipated — this is a one-time compliance pass. Future violations caught by POL-001 gate.
